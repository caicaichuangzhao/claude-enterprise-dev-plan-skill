# 项目启动前检查清单

## 一、需求调研（必填）

### 1. 语言与地区

- [ ] 是否需要多语言支持？
  - 当前计划支持语言：___
  - 后续可能增加的语言：___
- [ ] 是否需要 RTL (从右到左) 语言支持？(阿拉伯语、希伯来语等)
- [ ] 日期格式：按地区还是统一？
- [ ] 货币：实时汇率还是固定？
- [ ] 图片资源是否需要多语言版本？

### 2. 支付渠道

- [ ] 确认支付渠道清单：
  - □ 支付宝
  - □ 微信支付
  - □ Stripe
  - □ PayPal
  - □ 其他：___
- [ ] 是否需要沙箱测试模式？
- [ ] 是否需要开发者模式快速切换支付方式？
- [ ] 支付回调URL是否需要多环境配置？

### 3. 部署环境

- [ ] 单服务器 / 分布式 / 云原生？
- [ ] 是否需要 Docker 容器化？
- [ ] CI/CD 流程：___
- [ ] 是否需要多环境（dev/staging/prod）？

### 4. 终端形态

- [ ] 纯 Web 应用？
- [ ] 需要原生 App？(iOS/Android)
- [ ] 需要小程序？
- [ ] 需要桌面端？
- [ ] API 是否需要被多个客户端共享？

### 5. 数据规模

- [ ] 预估用户数：___
- [ ] 日活：___
- [ ] 数据量增长曲线：___
- [ ] 是否需要分表分库？

### 6. 合规要求

- [ ] GDPR 合规？
- [ ] 数据本地化要求？(数据必须存储在特定地区)
- [ ] 审计日志需求？
- [ ] 敏感数据加密方式：___

### 7. 第三方集成

- [ ] 需要对接的外部服务清单：
  - □ 认证服务：___
  - □ 支付服务：___
  - □ 文件存储：___
  - □ 消息服务：___
  - □ 其他：___
- [ ] 是否需要抽象层防止供应商锁定？

### 8. 开发者体验

- [ ] 是否需要开发者模式调试面板？
- [ ] 是否需要支付沙箱？
- [ ] 是否需要 API 调试工具？
- [ ] 是否需要性能分析工具？
- [ ] 日志级别控制？
